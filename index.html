Tentang CRUD

Pengertian Crud

Crud singkatan dari proses Create, Read, Update, dan Delete digunakan untuk mengelola data pada sistem atau aplikasi pengolah data. Ini biasanya digunakan dalam sistem informasi, aplikasi web, atau aplikasi mobile untuk mengatur interaksi dengan database.

Empat poin tersebut mengindikasikan bahwa fungsi utama melekat pada penggunaan database relasional beserta aplikasi yang mengelolanya, seperti Oracle, MySQL, SQL Server, dan lain - lain.

C R U D

  • Create
  • Read
  • Update
  • Delete

    • Fungsi Crud

    1. Create (Buat): Tahap pertama dalam proses CRUD adalah membuat data baru. Dalam tahap ini, data baru diinputkan ke dalam sistem atau aplikasi pengolah data. Data ini kemudian disimpan dalam database dengan mengisi nilai-nilai yang sesuai untuk setiap kolom atau atribut.

    2. Read (Baca): Pada tahap ini, sistem atau aplikasi membaca data yang ada dalam database. Data dapat diambil dan ditampilkan kepada pengguna atau digunakan untuk berbagai tujuan. Ini melibatkan membaca data dari satu atau beberapa tabel dalam database.

    3. Update (Perbarui): Tahap update melibatkan mengubah data yang sudah ada dalam database. Pengguna atau sistem dapat memperbarui nilai-nilai yang ada dalam record atau baris data tertentu. Ini diperlukan ketika ada perubahan atau koreksi yang perlu diaplikasikan pada data yang ada.

    4. Delete (Hapus): Tahap terakhir adalah menghapus data dari database. Jika data tidak diperlukan lagi atau sudah usang, pengguna atau sistem dapat menghapusnya. Hapus ini biasanya bersifat permanen, dan data yang dihapus tidak dapat dipulihkan kecuali ada proses pemulihan khusus yang diterapkan.
